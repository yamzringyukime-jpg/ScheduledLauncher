# ScheduledLauncher（定時起動アプリ）

## プロジェクト概要

Windowsタスクスケジューラと連携した定時起動アプリ。スリープ復帰時にも確実に実行され、タスクトレイ常駐でカレンダーUIによるON/OFF設定が可能。


## 目的・ターゲット・利用シーン

**目的:**
- 毎日決まった時間に特定のアプリやURLを自動で開きたいユーザー
- PCがスリープ中でも復帰時にタスクを実行したいユーザー
- ポータブルアプリとして持ち運びたいユーザー

**利用シーン:**
- 出勤時に自動で勤怠管理システムを開く
- 定時に業務用アプリを複数順次起動する
- 休日をカレンダーで設定して自動起動をスキップする


## 運用の前提条件

- Windows 10/11
- 管理者権限を持つユーザーアカウント（タスクスケジューラ登録に必要）
- Python 3.10以上（開発時）
- 配布時はPyInstallerでEXE化（ポータブル仕様）


## 仕様・要件

| 機能 | 内容 |
|------|------|
| ブラウザ自動起動 | 指定URLを通常/シークレットモードで開く（ブラウザ選択対応：default/chrome/edge/firefox） |
| 外部アプリ起動 | 任意のEXEファイルを実行 |
| 時間差順次起動 | 複数アプリを指定秒数間隔で順次開く |
| カレンダーUI | 1ヶ月表示で個別日のON/OFFを切り替え |
| タスクスケジューラ連携 | スリープ復帰時トリガー対応 |
| タスクトレイ常駐 | 設定変更、手動テスト、カレンダー表示 |

## Knowledge & Decisions (重要な意思決定)

### Reasoning

- **タスクスケジューラ方式を選択:** スリープ復帰時の確実な実行が要件であり、Python単体では不可能なため
- **ハイブリッド構成を選択:** カレンダーUIが必要なため、タスクスケジューラ単体では不足
- **PyInstaller単一ファイル（onefile）を選択:** 「EXEだけをコピーして持っていく」というユーザーの直感的な操作に合わせ、ファイルコピー漏れによる起動失敗リスクを排除するため。当初は起動速度や一時フォルダの問題から `onedir` 推奨としていたが、利便性とポータブルの確実性を優先した。

### Pitfalls

- **タスク削除の漏れ:** 異常終了時に残存する。対策として起動時クリーンアップと手動削除ツールを用意
- **UACプロンプト:** 非管理者ユーザーではタスク作成・削除に毎回プロンプトが出る。管理者権限ユーザー使用を前提とする
- **不完全なコピーによる起動失敗:** PyInstallerの `onedir` 形式でビルドされた EXE のみをコピーし、依存ライブラリが入ったフォルダ（`_internal` 等）を忘れると、読み込みエラーで即座にクラッシュする。これを防ぐために `onefile` ビルドを原則とする。
- **タスクスケジューラ起動と二重起動防止（Mutex）の衝突:** タスクトレイ常駐プロセスとタスクスケジューラからの実行プロセス（`--run`）は別プロセスになる。`main()`関数の先頭で一律にMutexチェックを行うと、常駐プロセスがいるためにスケジュール実行が即死してしまう（ログも残らない）。`--run` モードはMutexチェックから除外する必要がある。
- **Tkinterアンチパターン（複数回tk.Tk()生成）:** 現在、open_settings()、run_apps()、register_task()、quit_app()の各メソッドで個別にtk.Tk()を生成している。Tkinterは単一インスタンス前提であり、頻繁な生成・破棄でTcl/TkインタプリタがクラッシュしたりZombie Windowになるリスクがある。現在は不安定挙動未確認だが、今後問題が出た場合はpystrayをバックグラウンドスレッド、Tkinterをメインスレッドで常駐させるアーキテクチャへのリファクタリングを検討


## タスクスケジューラ手動削除方法

アプリが異常終了した場合、タスクが残存することがあります。以下のいずれかの方法で削除してください。

### 方法A: タスクスケジューラGUI

1. タスクスケジューラを開く（Win+R → taskschd.msc）
2. タスクスケジューラライブラリ → ScheduledLauncher を右クリック → 削除

### 方法B: コマンドプロンプト

```cmd
schtasks /delete /tn "ScheduledLauncher" /f
```

### 方法C: PowerShell

```powershell
Unregister-ScheduledTask -TaskName "ScheduledLauncher" -Confirm:$false
```

### 方法D: cleanup.exe

配布フォルダ内の `cleanup.exe` をダブルクリック
